What is Docling?

In the age of large language models and retrieval-augmented generation (RAG), the ability to accurately parse and extract structured data from documents has become essential. Docling, developed by IBM Research, is an open-source Python library that transforms complex documents—PDFs, DOCX files, images, and more—into clean, structured, AI-ready formats.

Unlike simple text extractors, Docling employs advanced AI models to understand document layout, identify tables, recognize headings, and preserve the semantic structure of content. This makes it an invaluable tool for building data pipelines, powering RAG systems, and automating document-heavy workflows.

💡 Key Insight: Docling goes beyond OCR and basic parsing. It uses deep learning models for layout analysis and table structure recognition, delivering structured output that captures the true hierarchy and meaning of documents.

Getting Started

Installation

Docling is available on PyPI and can be installed with a single pip command. For GPU support, additional dependencies may be required.

bash 
# Basic installation
pip install docling

# With OCR support
pip install docling[ocr]

# With GPU acceleration (CUDA)
pip install docling[gpu]

Basic Document Conversion

The simplest way to use Docling is through the DocumentConverter class, which handles the entire parsing pipeline in just a few lines of code.

Python 
from docling.document_converter import DocumentConverter

# Initialize the converter
converter = DocumentConverter()

# Convert a PDF document
result = converter.convert("research_paper.pdf")

# Access the parsed document
document = result.document

# Export as Markdown
markdown_output = document.export_to_markdown()
print(markdown_output)

# Export as JSON (structured)
json_output = document.export_to_dict()

AI Models Under the Hood

Docling uses a suite of specialized deep learning models, each trained for a specific aspect of document understanding. These models work together in a pipeline to produce accurate, structured output.

1. Layout Analysis Model

The layout analysis model identifies document regions such as titles, headings, paragraphs, images, tables, lists, and footnotes. It is based on object detection architectures and has been trained on large corpora of diverse document types.

"Accurate layout analysis is the foundation of document understanding. Without knowing where elements are and what type they are, downstream extraction becomes unreliable." — IBM Research

2. Table Transformer (TableFormer)

The TableFormer model is specifically designed for table structure recognition. It detects table boundaries, identifies rows and columns, handles merged cells, and reconstructs the complete table structure in a machine-readable format.

Python 
from docling.document_converter import DocumentConverter, PdfFormatOption
from docling.datamodel.pipeline_options import PdfPipelineOptions, TableFormerMode

# Configure pipeline with detailed table extraction
pipeline_options = PdfPipelineOptions(do_table_structure=True)
pipeline_options.table_structure_options.mode = TableFormerMode.ACCURATE

converter = DocumentConverter(
    format_options={
        "pdf": PdfFormatOption(pipeline_options=pipeline_options)
    }
)

result = converter.convert("financial_report.pdf")

# Access extracted tables
for table in result.document.tables:
    print(table.export_to_dataframe())

3. OCR Models

For scanned documents and images, Docling integrates with industry-standard OCR engines. The library supports both Tesseract (open source) and EasyOCR (deep learning based), with the ability to configure which engine to use on a per-document basis.

Python 
from docling.datamodel.pipeline_options import PdfPipelineOptions, OcrOptions

# Enable OCR for scanned documents
pipeline_options = PdfPipelineOptions(do_ocr=True)

# Configure OCR engine
pipeline_options.ocr_options = OcrOptions(
    engine="easyocr",      # or "tesseract"
    lang=["en", "de"],     # supported languages
)

RAG Pipeline Integration

One of Docling's strongest use cases is integration with Retrieval-Augmented Generation (RAG) pipelines. Docling provides built-in chunking strategies that produce optimized text segments for embedding and retrieval.

Using Docling with LangChain

Python 
from docling.document_converter import DocumentConverter
from docling.chunking import HybridChunker

# Parse the document
converter = DocumentConverter()
result = converter.convert("technical_manual.pdf")

# Chunk the document for RAG
chunker = HybridChunker(tokenizer="sentence-transformers/all-MiniLM-L6-v2")
chunks = list(chunker.chunk(result.document))

# Each chunk preserves metadata and structure
for chunk in chunks[:3]:
    print(f"Text: {chunk.text[:100]}...")
    print(f"Metadata: {chunk.meta}")
    print("---")

Batch Processing

Docling supports batch processing of multiple documents, making it suitable for enterprise-scale document pipelines.

Python 
from docling.document_converter import DocumentConverter
from pathlib import Path

converter = DocumentConverter()

# Process multiple documents
pdf_files = list(Path("./documents/").glob("*.pdf"))

results = converter.convert_all(pdf_files)

for result in results:
    doc = result.document
    print(f"Processed: {result.input.file.name}")
    print(f"  Pages: {doc.num_pages()}")
    print(f"  Tables found: {len(doc.tables)}")
    print(f"  Output: {doc.export_to_markdown()[:200]}...")

Advanced Configuration

Docling offers fine-grained control over every aspect of the document conversion pipeline. Here are the most commonly used configuration options:

Pipeline Options

Python 
from docling.document_converter import DocumentConverter, PdfFormatOption
from docling.datamodel.pipeline_options import PdfPipelineOptions

# Full configuration example
pipeline_options = PdfPipelineOptions(
    do_ocr=True,                  # Enable OCR for scanned pages
    do_table_structure=True,      # Enable table detection
    images_scale=2.0,             # Scale for image extraction
    generate_page_images=True,    # Generate page-level images
    generate_picture_images=True, # Extract embedded pictures
)

converter = DocumentConverter(
    format_options={
        "pdf": PdfFormatOption(pipeline_options=pipeline_options)
    }
)

result = converter.convert("complex_document.pdf")

Performance & Best Practices

  • Use GPU acceleration when processing large document batches. The AI models benefit significantly from CUDA-enabled hardware, reducing processing time by up to 10x.
  • Disable unnecessary pipeline stages (e.g., turn off OCR for native PDFs) to speed up processing when you know the input document type.
  • Leverage batch processing with convert_all() for multiple documents rather than processing them sequentially.
  • Choose the right table mode: Use TableFormerMode.FAST for quick extraction and TableFormerMode.ACCURATE when precision matters most.
  • Cache converted results when reprocessing the same documents, as the AI inference is the most compute-intensive step.
